简历与作品集

简历：一页优先

应届与实习简历建议控制在一页 A4（中文 PDF 为主）。突出与目标岗位相关的课程、项目、实习、技能，减少与求职无关的兴趣罗列。

项目描述怎么写

  • 一句话背景 + 你的职责 + 技术或方法 + 结果（数据、性能、用户量等）。
  • 避免空泛词：「负责开发」不如「实现 REST 接口 12 个，支撑日均请求约 xx」。
  • 课程大作业可写，但需说明个人贡献，避免整段复制小组报告。

Git 与作品集

  • 仓库保持 README：项目简介、如何运行、技术栈、截图或演示链接。
  • 提交记录尽量规律，避免「一次性上传」；可展示 code review 或 issue 协作。
  • 若岗位偏设计/产品，可补充 Figma、作品集站点等链接（确保可访问）。

使用本站解析简历

上传 PDF / DOCX / TXT 至 个人简历上传，可结合系统推荐岗位与报告做迭代修改。
